# 指数基金回撤分层策略记录

版本时间：2026-05-30 18:24:34  
策略版本：v2  
状态：草案，后续可继续迭代

## 1. 策略目标

本策略用于管理当前指数基金与 QDII 基金组合的加仓节奏。

核心目标：

- 用回撤替代单日涨跌作为主要触发依据
- 按资产波动特征设置不同阈值
- 给每个资产组设置独立预算
- 对科技成长类资产设置合并预算上限
- 用月度再平衡避免越跌越集中加仓
- 每日只生成策略信号，实际执行以月度预算为准

## 3. 资产组内基金分配模型

策略先对资产组计算信号，再按用户配置把加仓单位拆分到具体基金。

```text
第一步：资产组生成信号
例如：纳指100资产组触发 2U

第二步：根据组内分配比例拆到基金
例如：
270042：50%
040046：25%
019547：25%

第三步：生成具体基金加仓建议
270042：1U
040046：0.5U
019547：0.5U
```

组内分配比例暂不由策略自动决定，由用户后续手动配置。

配置模型：

```text
资产组：
    基金代码：
        分配比例：
        是否启用：
        单基金单日上限：
        单基金单月上限：
```

规则：

```text
同一资产组内，启用基金的分配比例合计应为 100%。
如果未配置组内分配比例，则只输出资产组级别建议，不拆分到具体基金。
如果某只基金暂停申购或被用户停用，则按剩余启用基金重新归一化分配。
```

## 4. 科技成长总暴露

```text

恒生科技
全球科技互联
```

因此除单资产组预算外，还需要设置科技成长总预算上限。

建议初始预算：

```text
纳指100：单月最多 4U
恒生科技：单月最多 4U
全球科技互联：单月最多 3U

科技成长合计：单月最多 7U
```

示例：

```text
纳指100触发 3U
恒生科技触发 3U
全球科技互联触发 2U

理论合计 8U
但科技成长总上限为 7U
因此本月最多执行 7U
```

## 5. 回撤计算

主信号使用 250 交易日高点回撤：

```text
回撤 = 当前指数点位 / 最近250个交易日最高点 - 1
```

每日计算指标：

```text
今日涨跌幅
20日涨跌幅
60日涨跌幅
250日高点回撤
当前回撤分层
60日均线状态
120日均线状态
250日均线状态
```

## 6. 回撤阈值分层

### 5.1 低波动资产

适用资产：

```text
沪深300
红利低波50
```

规则：

```text
250日高点回撤 0% ~ -5%        不额外加仓
250日高点回撤 -5% ~ -10%      触发 1U
250日高点回撤 -10% ~ -15%     触发 2U
250日高点回撤 -15% ~ -25%     触发 3U
250日高点回撤 < -25%          触发 3U，不再继续放大
```

### 5.2 中波动资产

```

250日高点回撤 -12% ~ -18%     触发 2U
250日高点回撤 -18% ~ -28%     触发 3U
250日高点回撤 < -28%          触发 3U，不再继续放大
```

### 5.3 高波动资产

适用资产：

```text

```

## 7. 趋势过滤

回撤触发后，根据均线位置调整实际执行单位。

```text
价格 > 60日均线：
    不执行，只提示观察

价格 <= 60日均线，且 > 120日均线：
    执行 50% 触发单位

价格 <= 120日均线，且 > 250日均线：
    执行 100% 触发单位

价格 <= 250日均线：
    执行 100% 触发单位
    但不因为跌破250日均线额外加码
```

示例：

```text
纳指100 250日回撤 -18%
按高波动规则触发 2U

如果当前价格仍高于60日均线：
实际执行 0U，只提示观察

如果价格低于60日均线但高于120日均线：
实际执行 1U

如果价格低于120日均线：
实际执行 2U
```

## 9. 月度再平衡

每日系统生成信号，并直接给出当日可执行加仓建议。  

执行节奏：

```text
每日：
    计算信号
    推送观察和建议

每月固定一次：  
    汇总本月所有触发信号  
    检查各资产是否超配或低配  
    校准下月预算与阈值  
```

建议月度执行日：

```text
每月第一个交易周的周三
```

理由：

- 避开月初和月底资金流扰动
- 避开多数节假日边界
- 不追求精确择时，强调纪律执行

## 10. 每日执行优先级

当多个资产同时触发，但总预算不够时，按优先级分配。  

```text
第一优先级：
    回撤达到第二档以上，且价格低于120日均线

第二优先级：
    回撤达到第一档，且价格低于120日均线

第三优先级：
    回撤达到第二档以上，但价格只低于60日均线

第四优先级：
    只触发观察，不执行
```

同优先级内，按回撤深度排序。

## 11. 全球科技互联代理规则

`006373 / 021842` 不按单一指数处理。

理想代理：

```text
40% MSCI全球信息科技指数
40% 中证海外中国互联网指数
20% 现金
```

如果第一版数据不好拿，可以先用简化代理：

```text
50% 纳斯达克100
30% 恒生科技
20% 现金
```

由于全球科技互联是主动 QDII 基金，不应完全按指数基金处理。第一版采用双层确认：

```text
第一层：代理指数回撤
第二层：基金自身净值回撤
```

基金自身回撤计算：

```text
基金自身回撤 = 当前基金净值 / 最近250个交易日基金净值高点 - 1
```

执行规则：

```text
代理指数回撤 0% ~ -10%      不加仓
代理指数回撤 -10% ~ -15%    观察，不直接加仓
代理指数回撤 -15% ~ -25%    触发 1U
代理指数回撤 < -25%         触发 2U
```

确认条件：

```text
基金自身250日回撤必须 <= -8%
否则只提示观察，不执行加仓
```

预算限制：

```text
全球科技互联单日最多：1U
全球科技互联单月最多：3U
全球科技互联参与科技成长合计预算
```

推送中必须明确标注：

```text
全球科技互联：使用简化代理，不等同真实基金持仓
全球科技互联：主动 QDII，使用降权执行规则
```

## 12. 每日推送格式

基金申购状态检查：

```text
主数据源：AKShare fund_purchase_em()
检查字段：申购状态、赎回状态、下一开放日、购买起点、日累计限定金额、手续费
执行位置：策略建议生成之后，飞书推送之前
```

规则：

```text
如果基金开放申购：保留策略建议
如果基金暂停申购：最终建议改为不执行
如果基金限制大额申购：按日累计限定金额修正最终建议
如果基金状态缺失或接口异常：推送中标记为“申购状态未确认”，不自动放大建议
```

推送时间规划：

```text
午间观察推送：北京时间 11:30 ~ 11:40
下午执行推送：北京时间 14:30 ~ 14:45
```

说明：

```text
午间观察推送用于提前了解风险和可能触发的资产，不作为最终加仓依据。
下午执行推送用于生成当日可执行加仓建议，优先服务场外基金北京时间15:00前的申购截止时间。
A股、港股、日股、美股相关基金统一纳入两次推送。
美股市场在北京时间凌晨收盘，国内白天推送时无法获取当日美股收盘，只使用上一美股交易日收盘数据。
```

## 13. 第二版核心规则总结

```text
主信号：250交易日高点回撤
辅助信号：60/120/250日均线
执行方式：每日提示，月度执行
预算方式：单资产组预算 + 科技成长总预算 + 全组合总预算
加仓方式：分层触发，最高档封顶，不无限放大
特殊资产：全球科技互联单独做代理，不并入纳指100
```
